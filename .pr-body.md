## Summary
Proximity alerts, location enablement, map controls, walking tours, and image pipeline cleanup for The Pilsen Mural Project.

## Highlights

### Location & proximity
- **Opt-in location**: Enable-location prompt after map load; choice (Enable / Not now) persisted in localStorage.
- **User on map**: Blue dot, 120 m geofence circle, GeolocateControl in map toolbar; zoom-to-user on first enable (pitch 50, zoom 16).
- **Nearby mural card**: When within 80 m, bottom card shows closest mural with distance, address, Get directions, Share; View/Dismiss advances queue; tour context (Stop N of M, next stop distance).

### Map UX
- **Pilsen boundary**: Neighborhood outline (Community Area 31) from Chicago Data Portal.
- **Fit map / Fit tour**: Header button fits bounds to all murals or tour stops.
- **Layer toggle**: Standard ↔ Satellite in map controls; custom layers re-applied after style change.
- **Controls**: Single toolbar (zoom, compass, fit, satellite) top-right below header.

### Walking tours
- **Collections**: `data/collections.json` with sample tours; tour list bottom sheet; active tour filters map/list to ordered stops.
- **Route line**: Amber LineString connecting stops; stop numbers on list and markers; "Leave tour" resets.

### Accessibility
- Skip link, focus trap/return in modal and sheets, focus-visible rings on interactive elements.

### Images & performance
- **Display/thumb only**: High-res removed; pipeline outputs WebP (display 1600px, thumb 400px); `high-res/` deleted.
- **srcset**: Modal, list, and markers use srcset/sizes; lazy loading and async decode.
- **Loading**: Full-viewport overlay with Pilsen boundary SVG progress until map ready.

### Mobile
- Safe areas, glass header, bottom sheets (MuralList, TourList), 44px touch targets.

## Refactor / cleanup
- Single React root for markers (createPortal); cluster when zoomed out; fanned deck for overlapping markers; zoom ≥16 disables grouping and spreads same-location markers in a circle.
- `image-stats.sh` script for image inventory.
